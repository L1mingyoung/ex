# AI Companion 项目规则

## 开发工作记录规则

每次进行完善或开发工作后，必须将变更内容写入以下两个文档：

1. **实施计划** `docs/Implementation_Plan.md` — 记录功能状态、完成/未完成对照、当前进度
2. **学习笔记** `docs/Learning_Notes.md` — 记录技术知识点、踩坑记录、设计决策理由

### 写入格式

#### 实施计划追加格式

```markdown
## YYYY-MM-DD 功能更新：简述

### 变更内容

| 变更项 | 说明 | 状态 |
|--------|------|------|
| 具体功能 | 简述 | ✅ / ⏳ / ❌ |

### 当前完成状态

| 子项 | 状态 | 说明 |
|------|------|------|
| ... | ✅ | ... |
```

#### 学习笔记追加格式

```markdown
## YYYY-MM-DD | 主题

### 技术要点

- 要点1
- 要点2

### 踩坑记录

- 问题描述 → 解决方案

### 设计决策

- 决策：xxx → 理由：xxx
```

## 前端开发规范

- 组件使用函数式组件 + Hooks
- 状态管理使用 Context + useReducer
- 样式使用 CSS 文件，不使用 CSS-in-JS
- 命名：组件 PascalCase，CSS 类 kebab-case，变量 camelCase
- 移动端适配使用 CSS 媒体查询，断点 768px
- 安全区域使用 `env(safe-area-inset-*)` CSS 变量
- 交互反馈：按钮 `:active` 缩放，Toast 替代 alert
