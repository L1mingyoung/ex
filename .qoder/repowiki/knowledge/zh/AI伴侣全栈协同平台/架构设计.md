- **容器化编排**：利用 `docker-compose.yml` 定义 `postgres`、`embedding`（Python/FastAPI）和 `api`（NestJS）三个核心服务，通过 `depends_on` 与健康检查（healthcheck）确保数据库与向量服务就绪后再启动主 API。
- **跨语言通信契约**：后端通过环境变量 `PYTHON_EMBED_URL` 动态发现并调用 Python 向量服务，实现 Node.js 与 Python 运行时的解耦协作。
- **共享类型系统**：在根目录维护 `shared/types.ts`，为 NestJS 后端、React 前端及小程序适配器提供零依赖的 TypeScript 接口定义，确保前后端数据交互的一致性。
- **统一构建入口**：通过 `package.json` 中的 `build:all` 脚本串联后端编译与前端打包流程，简化全栈应用的构建步骤。
