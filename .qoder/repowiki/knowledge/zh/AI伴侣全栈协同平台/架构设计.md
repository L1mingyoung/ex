- **容器化编排**：使用 `docker-compose.yml` 统一管理 PostgreSQL、Python向量化服务、NestJS后端及QQ机器人适配器，通过 `depends_on` 和 `healthcheck` 确保服务启动顺序。
- **跨语言契约**：通过 `shared/types.ts` 定义零依赖的 TypeScript 接口，作为前端（React）、后端（NestJS）及小程序适配器的通用数据契约，确保角色、会话及消息结构的一致性。
- **服务间通信**：后端通过环境变量 `PYTHON_EMBED_URL` 调用独立的 Python 向量化服务，并通过内部网络与数据库交互；适配器通过 `API_BASE` 访问后端 REST API。
- **配置集中管理**：根目录 `.env` 文件统一定义数据库凭证、API密钥及服务端口，通过 Docker 注入各子服务，实现环境隔离与配置同步。
