- 所有业务逻辑与状态操作均封装在 `AppProvider` 中，组件仅通过 `useAppContext` 消费状态与触发 Action，严禁在 UI 组件中直接调用 API。
- API 层采用纯函数导出方式，不依赖 React 上下文，确保 API 逻辑可在非浏览器环境（如小程序、Node.js）中复用。
- 流式响应处理统一在 `sendMessageStream` 中解析 SSE 格式，并通过 `AbortController` 支持用户主动中断请求。
- 组件拆分遵循单一职责原则，复杂区域（如 Sidebar, ChatArea）均拆分为独立的子组件文件，并通过 CSS 类名隔离样式。
